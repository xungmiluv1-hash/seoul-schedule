# 서울인력 스케줄 관리 시스템

## 프로젝트 개요
의료 통역사 자동 배정 시스템. 병원별 환자 스케줄을 붙여넣으면 통역사를 자동으로 배정한다.

**현재 버전:** v55 (2026-05-15)  
**기술 스택:** 순수 HTML + CSS + Vanilla JavaScript + Leaflet.js (지도)  
**배포:** GitHub → Vercel (https://github.com/xungmiluv1-hash/seoul-schedule, 단일 `index.html`)
**로컬 백업:** `seoul-schedule-v{N}.html` (Documents/seoul schedule project)  
**파일 크기:** ~300KB (v20 대비 -77%, bunny sprite 제거 후)

---

```
seoul-schedule/
```

```
브라운 ↔ 안과            : 0분 (같은 건물)
아트라인 ↔ 안과 ↔ 브라운 : 5분
```

### 언어 코드

```
V=베트남어   T=태국어    I=인도네시아어  U=우즈벡어
P=필리핀어   R/Ka=러시아어  EN/Sr=영어   MG/M=몽골어
MY=미얀마어  N=네팔어    CH=중국어      C=캄보디아어
KZ=카자흐어  KY=키르기스어
L=라우스어(태국어 통역)  K=한국어(통역 불필요)
AF=아프리카(영어 통역)
```

**언어 대체 규칙:**
- KZ/KY → U(우즈벡) 통역사 사용 가능
- 니어치과 U(우즈벡) 환자 → R(러시아) 통역사 대체 가능
- I/N 환자가 니어 아닌 병원에 있을 때 → EN으로 우선 배정 (니어용 native 보존)

**언어 코드 별칭 (v19+v22):**
- `MA`, `ML` (말레이시아) → I (인도네시아 통역사)
- `D` (동티모르) → I
- `L` (라오스/라우스) → T (태국 통역사)
- `AF`, `Af`, `af` (아프리카) → EN (영어 통역사) ← **v22 추가**

**우즈벡 통역사 슈퍼맨 배정 불가:**
- 닐료, 디아나, 리솔랏, 자리나, 나르샤 → 슈퍼맨비뇨기과 절대 불가 (종교적 사유, neverHosps 등록)
- 슈퍼맨 U/KZ/KY 환자 → R(러시아어) 또는 EN 대체

### 직원 규칙

#### 팀장 (한국어/영어 불가 시 연락)
- **황수미** (V): 명동(우선), 헤라, 니어 관리
- **한승미** (V+EN): 슈퍼맨, 다정한, 오아로, 아트라인 관리
- **홍주은** (V): 브라운, 안과, 닥터송 관리

#### 고정 배정
- **한미영** → 명동 T(태국) 최우선
- **최연이** → 다정한 T(태국) 최우선
- **아루니** → 아트라인 항상 고정 (다른 곳 배정 불가)
- **황수미** → 명동 V(베트남) 최우선
- **딘탄튀, 김가인, 한승미** → 다정한 V(베트남) 풀

#### 제한 규칙
- **유이리나5**: 브라운 절대 불가, 니어 선호
- **아스미타**: 브라운 절대 불가, 니어 선호
- **중순**: 수/금/토/일만 출근
- **닐료, 디아나, 리솔랏**: 브라운 선호 (preferBase), KZ/KY/R/U 모두 가능
- **아일린, 권마리, 에드나**: 브라운 선호 (필리핀어+영어)

#### 알바 규칙 (v19 업데이트)
- 알바는 각자 정해진 `days` 배열대로만 출근 (주말 한정 X)
- 평일에 출근하는 알바도 있음:
  - **항다르마** (MG/EN): 월/토/일
  - **카잉툰** (MY): 월/금/토/일
  - **자리나** (U): 화/금/토
  - **닌누이** (MY): 금/토
  - **헬레나** (I): 수/금
- 일요일은 해당 국가 환자가 있을 때만 호출 (기존 규칙 유지)

### 배정 인원 계산

#### 브라운 / 니어치과
팝업에서 신규/기존 환자 수 입력 후 결정:
- 기존 5명 이하 + 신규 없음 → 통역 1명
- 기존 5명 이하 + 신규 1명 이상 → 통역 2명 (신규 상담 시간↑)
- 기존 10명 이상 → 통역 2명
- 전체 6명 이상 → 통역 2명

#### 기타 병원
- 6명 이상 → 2명
- 6명 미만 → 1명

### 특수 상황

#### 슈퍼맨 / 아트라인 배정 불가 시
통역사가 없거나 모두 다른 곳에 있으면:
```
한국어/영어/전화통역가능여부확인하기-[언어명]환자
```
예: `한국어/영어/전화통역가능여부확인하기-미얀마환자`

#### 토요일 조기 마감
- 슈퍼맨 13:00, 다정한 14:00, 니어 15:00 마감
- 마감 후 브라운/니어/안과 등 이동 가능 여부 자동 계산
- 이동 가능하면: `끝나면 브라운 / 니어 이동 가능`
- 불가하면: `끝나면 연락주기`

#### 다정한 → 브라운 이동
- 90분 거리이므로 원칙적으로 불가
- 예외: 환자가 10시 또는 11시에 있으면 이동 가능
- **v22 엔진 구현:** `FAR_HOSPS = ['다정한','오아로']`. 이 지역 ↔ 다른 지역 이동은 far-side slot 시작 ≤ 11:00 일 때만 허용 (`FAR_MOVE_CUTOFF = 11*60`)

#### 최소 상담 시간 (v55 추가)
통역사는 마지막 환자 시각 + 최소 상담 시간 이후에만 다른 지점으로 이동 가능:
```
기본 (모든 병원)  : +30분 (최소 상담 시간)
다정한 / 오아로   : +60분 (1시간 이상 치료)
안과              : +120분 (시야검사 등 약 2시간 소요)
```
예: 니어 10:00 환자 → 통역사 free 10:30 → 슈퍼맨 10:30 이동 불가 (30분 travel 필요, 도착 11:00)

#### 토요일 출력 형식 (v55)
- 슈퍼맨 / 청량리(다정한) / 오아로: "끝나고 브라운 이동" (시간 미기재)
- 그 외 조기 마감 병원: "끝나면 브라운/니어 이동 가능" (기존 방식 유지)

#### 명동 팀장 우선 배정
- V(베트남) 환자 → 황수미 최우선
- T(태국) 환자 → 한미영 최우선
- 두 팀장 모두 다른 곳에 배정된 경우 → 일반 배정

## 입력 형식

```
브라운
V:9
C:13
U:4
T:15(10시/11시/2시)
R:2
P:10
N:1

니어
R:3
N:2(11시/2시)

다정한치과
T:5
V:3(오전)         ← 오전 = 10:00-12:00
V:2(오후)         ← 오후 = 13:00-17:00
MA:5              ← 말레이시아 → I 통역사

명동
V:2(11시/2시)
T:1(10시)

슈퍼맨
C:6               ← 시간 없음 = 전일 (allDay, 이동 불가)
V:2(10시/11시30분)
```

**시간 키워드 (v19):**
- `오전` → 10:00-12:00 (통역사는 13:00부터 이동 가능)
- `오후` → 13:00-17:00 (마감까지 근무)
- `출근` → 10:00 (체크인 시각)
- 시간 없음 → 전일 근무 (allDay), 다른 지점 이동 불가 (조기 마감 예외)

**병원명 인식 별칭:**
- 브라운 = 브라운치과 = 강남브라운
- 니어 = 니어치과 = 서울니어치과
- 다정한 = 다정한치과 = 청량리 = 청량리치과
- 헤라 = 청담헤라 = 청담헤라산부인과 = 청덤헤라
- 슈퍼맨 = 슈퍼맨비뇨기과

## 고정 배정 (특별 고정) — v19 multi-stop 지원

**일반 형식:**
```
황수미 → 명동
황수미 → 명동 10시/2시
```

**Multi-stop (하루 여러 지점 이동):**
입력 형식 자유 — 시간이 병원 앞/뒤 어느 위치에 있든 인식:
```
한승미 아트라인 출근 → 1시까지 니어 → 3시까지 브라운
한승미 아트라인 출근 → 니어 1시까지 → 브라운 3시까지
한승미 출근 아트라인 → 니어 1시까지 → 3시까지 브라운
```

**Output 출력 (출발지에만 표시, 다른 지점은 생략):**
```
아트라인
한승미 (10:00 · → 니어 1시까지 → 브라운 3시까지 이동 · 🔒고정)

니어
(한승미 표시 안 됨 — 출발지 아트라인에 이미 표시)

브라운
(한승미 표시 안 됨)
```

**Validation:**
- 오늘 스케줄에 없는 병원 → 자동 무시 + 경고 (`고정배정 무시: ...`)
- 우즈벡 통역사 → 슈퍼맨 lock 시도 → 무시 (neverHosps)

### 배정 알고리즘 (점수 기반)

낮은 점수 = 더 높은 우선순위:

```javascript
-20  // 오늘의 고정(Lock) 배정
-15  // alwaysFixed (아루니→아트라인)
-10  // fixedRules (한미영→명동T, 최연이→다정한T 등)
 -8  // 팀장 관리 병원
 -5  // preferBase (필리핀, 우즈벡 → 브라운)
  0  // 이미 해당 병원에 있음
  1  // 미배정 (처음 배정)
  2  // 다른 병원에서 이동 필요
999  // 불가 (neverBase, 중복, 이동 불가)
```

- `seoulLeave_v4`: 월간 휴무 데이터 (`{"2026-05-01": ["황수미"]}`)

## v19 핵심 규칙 (Engine 동작)

### allDay 규칙 (시간 미입력)
- 입력 `C:6` 처럼 시간 없는 환자 → 통역사는 해당 병원에 **전일 근무**, 다른 지점 이동 불가
- 예외: 해당 병원 조기 마감 + 마감시각 + 이동시간 ≤ 다음 지점 첫 환자 시각
- 데모 처리 순서: allDay 우선 처리 → 통역사 lock-in → 다른 demand 처리

### Multi-stop lock
- 입력: `한승미 아트라인 출근 → 니어 1시까지 → 브라운 3시까지`
- 파싱: `→` 또는 `->` 로 split, 각 segment에서 병원/시간/출근 키워드를 자유 순서로 추출
- 데이터: `locksByDate[dk]` 에 stop별 entry, 같은 `multiGroup` ID로 묶음
- Engine: `lockStops[name]` 으로 person→stops 추적, isLK는 모든 stop 매칭, anchor는 이전 stop
- Render: 출발지에만 chain 표시 (`→ 니어 1시까지 → 브라운 3시까지 이동`), 후속 지점은 생략

### 입력에 없는 병원에 대한 lock 무시
- `activeHospIds = parsed.map(p=>p.hospId)` 로 active 추출
- Lock의 hospId가 active에 없으면 무시 + 경고 추가
- 동일하게 standby의 `home = s.fixedBase` 도 active에 있을 때만 사용 (한미영 명동 fixedBase 버그 fix)

### 중복 배정 방지
- `assignedNames.has(s.name) && !(isLK && cur===d.hospId) && !canReuseAssigned` → sc=998 (불가)
- multi-stop은 예외 (각 stop이 동일 person의 다른 hospital)

## 개발 시 주의사항

### 파서 관련
- 병원명 인식: 공백 제거 후 소문자로 비교
- 언어 코드: 대소문자 무시, alias 테이블로 정규화
- 한국 콜론 `：`과 영문 콜론 `:` 모두 지원
- 파싱 순서: 병원명 먼저 → 이후 줄은 언어 코드로 처리

### 추가/수정 시 체크리스트
- [ ] 새 직원 추가: `STAFF` 배열에 객체 추가 (langs, days, fixedRules, neverBase 필수)
- [ ] 새 병원 추가: `HOSPITALS` 배열 + `HOSP_ALIASES` + `TRAVEL_TABLE` 업데이트
- [ ] 새 언어 추가: `LANG_ALIAS` + `LANGS` 업데이트
- [ ] 이동 시간 변경: `TRAVEL_TABLE` 배열 수정
- [ ] 고정 배정 변경: 해당 직원의 `fixedRules` 배열 수정

### 직원 객체 구조
```javascript
{
  name: '이름',
  langs: ['V', 'EN'],        // 가능 언어 코드 배열
  days: [0,1,2,4,5,6],      // 0=월, 1=화, 2=수, 4=금, 5=토, 6=일 (3=목 없음)
  type: 'jeong',             // 'jeong' | 'alba'
  fixedRules: [              // 우선 배정 규칙
    {hospId:'명동', lang:'V'}
  ],
  neverBase: false,          // true면 브라운 절대 불가
  preferNear: false,         // true면 니어 선호
  preferBase: false,         // true면 브라운 선호
  alwaysFixed: false,        // true면 fixedRules가 항상 최우선
  mgrHosps: ['헤라','니어'], // 팀장 관리 병원 (있을 때만)
}
```
