# 비즈니스 규칙 상세

## 1. 요일별 운영 규칙

| 요일 | 특이사항 |
|---|---|
| 월 (0) | 정직원만, 알바 없음 |
| 화 (1) | 정직원만 |
| 수 (2) | 정직원만. 다정한치과 휴진 |
| 목 (3) | **전체 휴무** — 모든 병원 휴진 |
| 금 (4) | 정직원만 |
| 토 (5) | 정직원 + 알바 출근 |
| 일 (6) | 정직원 + 알바 (해당 국가 환자 있을 때만 호출) |

## 2. 통역사 배정 우선순위 (상세)

### 2-1. 명동 특별 규칙
- 베트남 환자 → **황수미** 최우선 (fixedRule)
- 태국 환자 → **한미영** 최우선 (fixedRule)
- 두 팀장이 다른 곳 배정된 경우 → 일반 통역사 배정

### 2-2. 다정한치과 특별 규칙
- 태국 환자 → **최연이** 최우선
- 베트남 환자 → **한승미, 딘탄튀, 김가인** 풀에서 선택

### 2-3. 아트라인 특별 규칙
- **아루니** 항상 고정 (alwaysFixed=true)
- 아루니 휴무 시에도 다른 통역사 배정하되 메모 추가

### 2-4. 팀장 관리 병원 우선
팀장은 자신의 관리 병원에 베트남 환자가 있을 때 우선 배정:
- 황수미 → 헤라, 니어
- 한승미 → 슈퍼맨, 다정한, 오아로, 아트라인
- 홍주은 → 브라운, 안과, 닥터송

규칙: 1주일에 최소 1회 관리 병원 방문 (시스템에서는 기회가 있을 때 우선 배정으로 구현)

## 3. 이동 규칙 (상세)

### 3-1. 이동 가능 판단 기준 (v55 업데이트)
```
이동 가능 = (마지막 환자 시각 + 최소 상담 시간) + 이동시간 ≤ 다음병원_첫번째환자시간
```

**최소 상담 시간 (CONSULT_MIN):**
| 병원 | 최소 상담 시간 | 이유 |
|---|---|---|
| 안과 | 120분 (2시간) | 시야검사 등 장시간 검사 |
| 다정한치과 / 오아로 | 60분 (1시간) | 원거리 지점, 충분한 치료 시간 필요 |
| 기타 모든 병원 | 30분 | 최소 1:1 상담 시간 |

**예시:**
- 니어 10:00 환자 → 통역사 free 10:30 → 슈퍼맨(30분 거리) 도착 11:00 → 슈퍼맨 10:30 이동 **불가**
- 니어 10:00 환자 → 슈퍼맨 11:00 이동 **가능**

### 3-2. 다정한치과/오아로 특례 (FAR_HOSPS)
- 청량리(다정한)/오아로 ↔ 다른 지점: far-side slot 시작 ≤ 11:00 일 때만 허용
- 이동 시: 마지막 환자 + 60분 상담 + 90분 이동 = 최소 150분 후 도착

### 3-3. 토요일 조기 마감 처리
마감 후 이동 메모 자동 추가:

| 병원 | 출력 형식 |
|---|---|
| 슈퍼맨, 다정한, 오아로 | "끝나고 브라운 이동" (시간 미기재) |
| 니어 (15:00 마감) | "끝나면 브라운/안과 이동 가능" |
| 기타 조기 마감 | "끝나면 [병원명] 이동 가능" 또는 "끝나면 연락주기" |

## 4. 환자 수 규칙 (v69 업데이트)

### 4-1. 브라운 (예외 — 대형 거점)
| 언어 | 환자 수 | 통역사 수 |
|---|---|---|
| V / T | 10명 이상 | 3명 |
| V / T | 6~9명 | 2명 |
| V / T | 5명 이하 | 1명 |
| 기타 | 10명 이상 | 2명 |
| 기타 | 9명 이하 | 1명 |

### 4-2. 니어치과 (v69: MAX1 지점 추가)
- **언어당 최대 1명** (환자 수 무관)
- R + U/KZ/KY 합산 처리 → R 통역사 1명으로 전부 커버
- 태국어 / 베트남어 2명 배정 절대 불가 (특별배정 제외)

### 4-3. MAX1 병원 (언어당 항상 1명)
`MAX1_LANG_HOSPS = ['슈퍼맨','명동','헤라','닥터송','아트라인','오아로','니어']`
- 환자 수 무관, 언어당 통역사 1명
- 직원정보 수정 시 비밀번호 필요 (`seoul`)

### 4-4. 기타 병원
| 요일 | 환자 수 | 통역사 수 |
|---|---|---|
| 평일 | 10명 미만 | 1명 |
| 평일 | 10명 이상 | 2명 |
| 주말 | 6명 미만 | 1명 |
| 주말 | 6명 이상 | 2명 |

## 5. 언어 대체 규칙 (v68~v69 업데이트)

| 원래 언어 | 대체 / 처리 | 조건 |
|---|---|---|
| **U/KZ/KY at 니어** | **R 통역사 1명으로 합산 커버** | **니어 U 환자 전원 러시아어 가능 (v69 핵심)** |
| KZ, KY, KG | R (러시아) | 항상 — KG는 키르기스 국가코드 |
| La, L | T (태국) | 항상 — 배정 완료 시 누락 경고 없음 |
| I, N | EN | 니어 아닌 곳 + 니어에도 I/N 수요 있을 때 native 보존 |
| C (캄보디아) | — | 니어 배정 절대 불가 |
| 모든 언어 | EN | 최후 수단 |

### 러시아/우즈벡 그룹 배분 원칙 (v69)
**닐료 · 에바(리솔랏) · 유이리나** — 같은 지점 중복 배정 절대 불가

| 통역사 | 언어 | 선호 지점 | 제한 |
|---|---|---|---|
| 유이리나5 | R | 니어 우선 | 브라운 절대 불가, 다정한/오아로 절대 불가 |
| 에바(리솔랏) | U/R/KZ/KY | 브라운/다정한 | 슈퍼맨 절대 불가 |
| 닐료 | U/R/KZ/KY | 브라운 | 슈퍼맨 절대 불가 |

배분 흐름:
1. 헤라-R 수요 → 에바 또는 닐료 배정 (유이리나는 니어 우선이므로 헤라에서 점수 낮음)
2. 니어-R+U 합산 → 유이리나 배정 (preferNear 보너스 -4점)
3. 브라운 등 나머지 → 닐료 또는 에바

## 6. 알바 호출 규칙

### 평일
- 알바 호출 없음 (days 배열에 평일 없음)

### 토요일
- 출근 가능한 알바 모두 출근

### 일요일
- 해당 언어 환자가 있을 때만 호출
- 예: 일요일에 I(인도네시아) 환자 없으면 헬레나 호출 없음

## 7. 배정 불가 시 메모

### 슈퍼맨, 아트라인 배정 불가
통역사를 배정할 수 없을 때:
```
한국어/영어/전화통역가능여부확인하기-[언어명]환자
```
예시:
- 미얀마 환자 → `한국어/영어/전화통역가능여부확인하기-미얀마환자`
- 몽골 환자 → `한국어/영어/전화통역가능여부확인하기-몽골어환자`

### 기타 병원 배정 불가
- `⚠️ [언어] N명 배정 불가` 경고 표시

## 8. 브라운치과 기본 배치

브라운은 기본 거점으로:
- 우즈벡(닐료, 디아나, 리솔랏): 브라운 선호
- 필리핀(아일린, 권마리, 에드나): 브라운 선호 (영어 응대)
- 유이리나5: 브라운 절대 불가
- 아스미타: 브라운 절대 불가

## 9. 일일 로테이션

같은 날 같은 점수의 통역사 중 선택 시:
- **날짜 기반 시드(seed)로 셔플** → 매일 다른 순서
- 같은 날은 항상 같은 결과
- 고정 규칙(fixedRules) 있는 통역사는 로테이션 대상 제외

## 11. 직원 정보 보안 (v68)

직원·알바 정보 수정(추가/삭제/복원) 시 비밀번호 입력 필요:
- 비밀번호: `seoul`
- 세션 내 1회 입력 후 유지 (페이지 새로고침 시 재잠금)
- 뱃지 `🔒 잠김` / `🔓 수정가능` 으로 상태 표시
- 다른 사용자는 열람만 가능, 수정 불가
